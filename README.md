## Базовый код

#### Класс Api
Содержит в себе базовую логику отправки запросов. В конструктор передается базовый адрес сервера и опциональный объект с заголовками запросов.

Поля:

- `readonly baseUrl: string` - базовый url для api
- `protected options: RequestInit`- объект с настройками для формирования запроса.
- 
Параметры в конструкторе:

- `baseUrl: string` - базовый url для api
- `options: RequestInit` - объект с настройками для формирования запроса.

Методы:

- `protected handleResponse(response: Response): Promise<object>` - обрабатывает ответа с сервера. Если ответ с сервера пришел, то возвращается его в формате json, в противном случае формирует ошибку
- `get(uri: string): Promise<object>` - выполняет GET запрос на переданный в параметрах ендпоинт и возвращает промис с объектом, которым ответил сервер
- `post(uri: string, data: object, method: ApiPostMethods = 'POST'): Promise<object>` - принимает объект с данными, которые будут переданы в JSON в теле запроса, и отправляет эти данные на ендпоинт переданный как параметр при вызове метода. По умолчанию выполняется POST запрос, но метод запроса может быть переопределен заданием третьего параметра при вызове.

#### Класс EventEmitter

Брокер событий позволяет отправлять события и подписываться на события, происходящие в системе. Класс используется в презентере для обработки событий и в слоях приложения для генерации событий.

Поля:

- `_events: Map<EventName, Set<Subscriber>>` - хранит события в виде Map, где ключом является строка или регулярное выражение, а значением сет коллбэков.
- 
Методы, геттеры и сеттеры:

- `on<T extends object>(eventName: EventName, callback: (event: T) => void)` - подписка на событие
- `off(eventName: EventName, callback: Subscriber)` - отписка от события
- `emit<T extends object>(eventName: string, data?: T)` - инициализация события
- `onAll(callback: (event: EmitterEvent) => void)` - подписаться на все события (выполнить коллбек на любое событие)
- `offAll()` - для сброса событий
- `trigger<T extends object>(eventName: string, context?: Partial<T>)` - возвращает функцию, при вызове которой инициализируется требуемое в параметрах событие.

# Файл `basketData.ts`

## Описание

Файл `basketData.ts` содержит реализацию класса `ShoppingCart`, который управляет данными корзины покупок. Этот класс реализует интерфейс `IBasketData` и использует систему событий для оповещения о состоянии корзины, изменении её содержимого, а также для взаимодействия с другими частями приложения.

## Класс `ShoppingCart`

### Свойства

- `_goods`: Приватное свойство, содержащее массив товаров в корзине (`IProductItem[]`).
- `events`: Экземпляр класса `IEvents`, используемый для управления событиями в корзине.

### Конструктор

#### `constructor(events: IEvents)`

Создаёт экземпляр класса `ShoppingCart`, инициализируя массив товаров `_goods` пустым массивом и принимая объект `events` для управления событиями.

### Методы

#### `get goods(): IProductItem[]`

Геттер для получения текущего списка товаров в корзине.

**Возвращает:**

- Массив товаров (`IProductItem[]`).

#### `set goods(items: IProductItem[]): void`

Сеттер для установки списка товаров в корзине. Обновляет свойство `_goods` и вызывает метод `checkEmpty()` для проверки состояния корзины.

**Параметры:**

- `items`: Массив товаров (`IProductItem[]`).

#### `isInBasket(id: string): boolean`

Проверяет, находится ли товар с указанным `id` в корзине.

**Параметры:**

- `id`: Идентификатор товара (`string`).

**Возвращает:**

- `true`, если товар присутствует в корзине, иначе `false`.

#### `checkBasket(id: string): void`

Проверяет наличие товара в корзине по его `id` и, если товар уже в корзине, генерирует событие `addToBasket:disabled`.

**Параметры:**

- `id`: Идентификатор товара (`string`).

#### `addToBasket(item: IProductItem): void`

Добавляет новый товар в корзину, если его там ещё нет. Обновляет состояние корзины и генерирует событие `basketData:changed`.

**Параметры:**

- `item`: Объект товара (`IProductItem`).

#### `removeFromBasket(id: string): void`

Удаляет товар из корзины по его `id`. Обновляет состояние корзины и генерирует событие `basketData:changed`.

**Параметры:**

- `id`: Идентификатор товара (`string`).

#### `clearBasket(): void`

Полностью очищает корзину. Удаляет все товары, обновляет состояние корзины и генерирует событие `basketData:changed`.

#### `getGoodsNumber(): number`

Возвращает количество товаров в корзине.

**Возвращает:**

- Количество товаров (`number`).

#### `getTotal(): number`

Возвращает общую стоимость товаров в корзине.

**Возвращает:**

- Общую стоимость товаров (`number`).

#### `getIdsOfGoods(): string[]`

Возвращает массив `id` всех товаров, находящихся в корзине.

**Возвращает:**

- Массив идентификаторов товаров (`string[]`).

### Приватные методы

#### `checkEmpty(): void`

Проверяет, пуста ли корзина. Если корзина пуста, генерирует событие `basket:empty`, иначе — событие `basket:has-items`.

## Зависимости

- `IBasketData`: Интерфейс, который описывает структуру данных корзины.
- `IProductItem`: Интерфейс, описывающий свойства товара, который может быть добавлен в корзину.
- `IEvents`: Интерфейс для работы с событиями, используется для оповещения о состоянии и изменениях в корзине.

# Файл `orderInfo.ts`

## Описание

Файл `orderInfo.ts` содержит реализацию класса `Order`, который управляет данными заказа. Этот класс работает в связке с корзиной покупок, реализованной в классе `ShoppingCart`, и позволяет получить полную информацию о заказе, включая способ оплаты, контактные данные, адрес доставки и перечень товаров.

## Класс `Order`

### Свойства

- `cart`: Объект класса `ShoppingCart`, представляющий корзину покупок. Используется для получения данных о товарах в заказе и расчета общей стоимости.
- `payment`: Способ оплаты (`PaymentMethod`). По умолчанию установлен на `'cash'`.
- `email`: Электронная почта покупателя (`string`). По умолчанию пустая строка.
- `phone`: Номер телефона покупателя (`string`). По умолчанию пустая строка.
- `address`: Адрес доставки (`string`). По умолчанию пустая строка.

### Конструктор

#### `constructor(cart: ShoppingCart, payment: PaymentMethod = 'cash', email: string = '', phone: string = '', address: string = '')`

Создаёт экземпляр класса `Order`.

**Параметры:**

- `cart`: `ShoppingCart` — объект корзины покупок, на основе которого создаётся заказ.
- `payment`: `PaymentMethod` — способ оплаты (по умолчанию `'cash'`).
- `email`: `string` — электронная почта покупателя (по умолчанию пустая строка).
- `phone`: `string` — номер телефона покупателя (по умолчанию пустая строка).
- `address`: `string` — адрес доставки (по умолчанию пустая строка).

### Методы

#### `getOrderFullInfo(): IOrder`

Возвращает полную информацию о заказе в виде объекта, реализующего интерфейс `IOrder`.

**Возвращаемое значение:**

- Объект с полями:
  - `payment`: `PaymentMethod` — способ оплаты.
  - `email`: `string` — электронная почта покупателя.
  - `phone`: `string` — номер телефона покупателя.
  - `address`: `string` — адрес доставки.
  - `total`: `number` — общая стоимость товаров в корзине.
  - `items`: `string[]` — массив идентификаторов товаров из корзины.

## Зависимости

- `ShoppingCart`: Класс, реализующий корзину покупок, используется для хранения и обработки данных о товарах.
- `IOrder`: Интерфейс, описывающий структуру данных заказа, включая способ оплаты, контактные данные и перечень товаров.
- `PaymentMethod`: Тип, представляющий возможные способы оплаты (например, `'cash'`, `'creditCard'` и т.д.).

- **`events.on('success:submit')`**: Обработчик события, который закрывает модальное окно после успешного завершения заказа.

### Инициализация приложения

- **`loadProducts()`**: Вызов функции для загрузки данных о товарах при запуске приложения.

## Зависимости

- **`EventEmitter`**: Компонент для управления событиями.

- **`ProductItem`, `ShoppingCart`, `Order`**: Компоненты для управления данными о товарах, корзине и заказах.

- **`Page`, `ProductPreview`, `Basket`, `FormOrder`, `FormContacts`, `VSuccess`, `CardModal`, `Modal`**: Компоненты представлений для отображения различных частей интерфейса.

- **`AppApi`**: Компонент для взаимодействия с сервером через API.

- **`cloneTemplate`, `ensureElement`**: Утилиты для работы с DOM и шаблонами.
